Where Do Compilers Use Finite Automata?

 Posted on October 11, 2019  |  7 minutes

Finite Automata… What a fancy name! Formally, they are computational machines with a finite number of states. For instance, a digital watch counting only hours and minutes is a finite state machine, because it can only in one out of 1440 states (24 “hour state” times 60 “minute state” for each hour). A washing machine can be seen as a finite state machine too. It starts in the “Ready” state, then goes onto the “Washing” state, followed by the “Drying” state. [Read More]

What Is a Programming Language Parser?

 Posted on October 2, 2019  |  13 minutes

First of all, a correction: Parsing algorithms are not made for programming languages; they are made for language Grammars. A Grammar is a set of rules that specify how you can write characters one after the other in order to form valid sentences (that is, valid for that grammar). For example, the English grammar tells us that you can put the characters ‘y’, ‘o’, ‘u’ one after the other and form a valid token (the word “you”), and that if you also add the characters ‘ ‘, ‘a’, ‘r’, ‘e’, ‘ ‘, ‘g’, ‘r’, ‘e’, ‘a’, ’t’, you will have made a valid sentence (that is, “you are great”).
